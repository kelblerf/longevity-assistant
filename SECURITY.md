# Security Policy

## Reporting a vulnerability

If you find a security issue, please do not publish it in a public GitHub issue.

Report it privately to the repository owner and include:

- what is affected
- how the issue can be reproduced
- possible impact
- suggested mitigation, if known

## Sensitive data

Do not include any of the following in public issues, pull requests, or screenshots:

- API keys or tokens
- Notion workspace identifiers that should stay private
- personal health data
- biomarker exports
- genetics data
- private notes or conversation history

## Repository scope

This repository is for application code, architecture, and product documentation.
Production secrets and private user data must stay outside version control.

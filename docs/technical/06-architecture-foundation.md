# Architecture Foundation

## Main principle

Architektura musi byt od zacatku rozhodnuta tak, aby:

- neprerustala pevny jadrovy model
- oddelovala vrstvy znalosti a autority
- umela rozsirovat zdroje bez prekopani jadra

## System layers

### Identity layer

- user profile
- assistant rules
- goals
- constraints
- preferences

### UBZ philosophy layer

- UBZ framework
- breathwork
- fasting context
- rhythm and regeneration
- behavioral guidance

### Health layer

- biomarkers
- symptoms
- sleep
- energy
- stress
- movement
- nutrition

### Genetics layer

- DNA markers
- predispositions
- interpretations
- conflict hints

### Evidence/Research layer

- NotebookLM
- biomarker references
- curated medical sources
- web evidence

### Knowledge layer

- Notion core sources
- OneNote reflection sources
- extended Notion sources
- local files
- external drive files

### Guidance layer

- morning brief
- daily guidance
- evening reflection
- routine engine

### Action layer

- save summary
- create follow-up
- reminders
- later integrations

## Source-scope engine

### Default mode

- core + smart expansion

### Scope controls

- core only
- core + domain
- broader second brain
- local files
- external drive
- web

### Dynamic authority

- core truth for identity and rules
- evidence layer for medical and laboratory claims
- UBZ layer for rhythm, breath, behavior, motivation and regeneration
- extended sources as contextual, topic-dependent support

## Required contracts

Every important answer must expose:

- selected scope
- source categories used
- evidence basis
- UBZ basis
- DNA basis
- final recommendation

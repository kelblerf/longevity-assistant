# Implementation Roadmap

## Phase 1: Core foundation

- project scaffold
- shared contracts
- profile model
- assistant rules
- conversation model
- source attribution format

## Phase 2: Core knowledge

- Notion core connector
- OneNote connector
- document ingestion
- retrieval over core sources

## Phase 3: Guidance engine

- daily guidance flow
- morning brief flow
- evening reflection flow
- routine reminders model

## Phase 4: Evidence and DNA

- biomarker interpretation layer
- NotebookLM research layer
- genetics model
- DNA conflict logic
- nutrition conflict logic

## Phase 5: Extended source-scope

- broader Notion scopes
- local file indexing
- external drive source groups
- scope controls and expansion rules

## Phase 6: Voice and action flows

- speech input adapter
- speech output adapter
- save to Notion
- follow-up creation
- reminder engine

## Success condition

First stable implementation is successful when the assistant can:

- know the user profile
- search core sources
- combine UBZ and evidence correctly
- use DNA only as a recommendation signal
- explain source scope
- guide the day
- save outputs back into the system
